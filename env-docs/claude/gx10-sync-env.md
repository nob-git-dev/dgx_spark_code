# GX10 環境ドキュメント同期

ユーザーが自然言語で伝えた環境変更を `~/projects/docs/` のドキュメントに反映する。

## 引数

$ARGUMENTS — 環境に加えた変更の説明（自然言語）

## 手順

以下の手順を厳密に守ること。

### 1. 変更内容の分析

ユーザーの説明（$ARGUMENTS）と、この会話中の作業履歴から、以下を判断する：

- **どのファイルが影響を受けるか** を特定する（1つ以上）:
  - `hardware.md` — HW仕様の変更（ドライバ更新、ストレージ追加等）
  - `services.md` — サービスの追加・削除・設定変更（ポート、モデル、コンテナ等）
  - `network.md` — IP変更、SSH設定、新しいエンドポイント
  - `policies.md` — 開発ルール・制約の変更
- 影響を受けないファイルは一切読まない（トークン節約）

### 2. 対象ファイルの読み取りと編集

影響を受けるファイルのみを Read で読み取り、Edit で必要な箇所だけを更新する。

編集ルール：
- 既存の書式・構造を維持する
- 追加・変更・削除のいずれかを最小限の範囲で行う
- 推測で情報を追加しない — ユーザーの説明と会話履歴に基づく事実のみ
- 不明な点があればユーザーに確認する

### 3. 差分の提示と確認

編集が完了したら、以下の形式でユーザーに差分を提示する：

```
## 変更内容

### [ファイル名]
- [変更の要約]

確認のため `git diff` を表示します。
```

そして `cd ~/projects/docs && git diff` を実行して差分を表示する。

**ユーザーの承認を得るまでコミットしない。**

### 4. コミット

ユーザーが承認したら、以下を実行する：

```bash
cd ~/projects/docs && git add -A && git commit -m "[変更の要約（日本語、1行）]"
```

コミットメッセージは変更内容を簡潔に表す日本語とする（例: 「SGLangのモデルをQwen3-32Bに変更」）。

## 注意事項

- `snapshots/` は編集しない（スナップショットは手動取得のみ）
- `README.md` は構造変更がない限り編集しない
- 会話中に実際に行った変更のみを反映する（予定や計画は含めない）
